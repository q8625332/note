
> 部署 1.30为例

apt 源里没有 Kubernetes 官方软件源，所以找不到 `kubelet/kubeadm/kubectl`。不要用 snap 安装，建议添加 Kubernetes 官方 apt 源后再安装。


```
apt update
apt install -y apt-transport-https ca-certificates curl gpg
```

添加 Kubernetes 官方 GPG key：

```
mkdir -p /etc/apt/keyrings

curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.30/deb/Release.key \
  | gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
```

添加 Kubernetes apt 源：


```
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /' \
  > /etc/apt/sources.list.d/kubernetes.list
```

更新 apt 缓存：

```
apt update
```

然后安装：

```
apt install -y containerd kubelet kubeadm kubectl
```

锁定版本，避免自动升级：

```
apt-mark hold kubelet kubeadm kubectl
```

检查：

```
kubeadm version
kubectl version --client
kubelet --version
```

初始化 Kubernetes，还需要处理 containerd 和 swap，例如：

```
swapoff -a
sed -i '/ swap / s/^/#/' /etc/fstab
```

生成 containerd 默认配置：

```
mkdir -p /etc/containerd
containerd config default > /etc/containerd/config.toml
```
启用 systemd cgroup：

```
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml
```
重启 containerd：

```
systemctl restart containerd
systemctl enable containerd
```

Master 执行：

```
kubeadm init --apiserver-advertise-address=192.168.1.10 --pod-network-cidr=10.244.0.0/16
kubectl apply -f https://github.com/flannel-io/flannel/releases/latest/download/kube-flannel.yml
```

Worker 执行：

```
kubeadm join 192.168.1.10:6443 --token xxx --discovery-token-ca-cert-hash sha256:xxx
```

最后 Master 检查：

```
kubectl get nodes
```
